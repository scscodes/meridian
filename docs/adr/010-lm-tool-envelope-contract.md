# ADR 010 — LM Tool Envelope Contract

**Status:** Superseded by [ADR 012](./012-product-reanchor.md) (2026-05-18) — the entire LM-tool surface was removed in the 2.0 re-anchor.
**Date:** 2026-04-20

## Context

LM tools previously returned ad-hoc text generated by command-specific formatter logic. This caused drift between command payloads and summaries, inconsistent behavior across commands, and weak guarantees for downstream consumers.

Foundation #3 requires a single typed contract so adding new tools is schema-first, not formatter-first.

## Decision

Adopt a canonical LM tool envelope for all tool invocations:

- `summary: string`
- `data: unknown`
- `followups: string[]`
- `renderHint: "status" | "chat_markdown" | "output_channel" | "tree_view" | "webview"`

Normalization is centralized in `src/infrastructure/lm-envelope.ts` with input:

- `commandName: CommandName`
- `result: Result<unknown>`

and output:

- `LmToolEnvelope`

`src/ui/lm-tools.ts` returns the serialized envelope as tool output for every command in `LM_TOOL_DEFS`.

## Rules

1. Domain handlers remain unchanged and continue returning `Result<T>` (ADR 002).
2. No VS Code API concerns are introduced into domain logic.
3. LM tool mappings are explicit for all registered LM tools; missing mappings fail fast.
4. Error envelopes use a deterministic structure in `data`:
   - `code`
   - `message`
   - optional `details`
   - optional `context`
5. `renderHint` semantics align with ADR 006 output-surface intent.

## Compatibility And Evolution

- Contract evolution is additive-only within V1 semantics.
- Existing chat and presentation paths accept either raw delegated payloads or envelope payloads (`envelope.data`) during transition.
- If a future breaking envelope change is needed, introduce explicit schema versioning in the envelope.

## Consequences

- LM tool consumers receive one stable shape across all domains.
- Formatter drift is reduced because normalization is centralized.
- Command onboarding requires explicit envelope mapping and tests, improving reliability.
